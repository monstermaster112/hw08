# 垃圾分类智能识别系统 — 项目报告

## 1. 选题与需求分析

### 1.1 解决什么问题

随着中国各地垃圾分类政策的全面推行，居民在日常生活中经常面临「这是什么垃圾」的困惑。不同城市、不同小区的分类标准虽有统一框架（可回收物、有害垃圾、厨余垃圾、其他垃圾），但具体物品的归类仍存在大量模糊地带：

- 大棒骨和鸡骨头都是骨头，却分属不同类别
- 沾了油污的外卖盒不属于可回收物
- 粽叶因为质地坚硬属于其他垃圾而非厨余垃圾

本项目的核心目标是：**提供一个便捷、直观、支持语音交互的垃圾分类查询工具**，帮助用户快速准确地判断任意物品的垃圾类别。

### 1.2 AI 角色

本项目中的 AI 体现在三个层面：

1. **语音识别层**：利用浏览器原生的 Web Speech API (webkitSpeechRecognition) 将用户语音转为文字，实现「说话即可查询」。
2. **自然语言理解层**：对用户输入进行语义解析，自动提取物品名称。支持「塑料瓶是什么垃圾」「电池属于哪类垃圾」等多种自然问法。
3. **规则推理层**：当物品不在数据库中时，通过关键词规则引擎推断其可能的分类。

### 2.2 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 前端界面 | HTML5 + CSS3 + 原生 JavaScript | 零依赖，双击即用 |
| 语音识别 | Web Speech API | 浏览器原生，无需 API Key |
| 语音合成 | SpeechSynthesisUtterance | 浏览器内置 TTS |
| CLI 工具 | Python 3.8+ 标准库 | 无需第三方依赖，跨平台 |
| 规则引擎 | Python re 模块正则匹配 | 轻量高效，可扩展 |

**为什么不用外部 AI API？** 轻量离线工具定位，避免网络依赖和隐私问题。

## 3. 关键实现说明

### 3.1 三段式分类识别引擎

核心算法采用三段式递进匹配：

```
输入: 物品名称
  |
  v
[第一段] 精确匹配
  +--- 在 ITEMS 字典中直接查找 key
  +--- 命中 -> 返回分类 + 描述
  +--- 未命中 -> 进入第二段
  |
  v
[第二段] 模糊匹配
  +--- 检查名称包含关系（name in key or key in name）
  +--- 命中 -> 返回匹配项的分类
  +--- 未命中 -> 进入第三段
  |
  v
[第三段] 规则引擎
  +--- 遍历 60+ 条关键词正则规则
  +--- 命中 -> 返回推断分类 + 规则依据
  +--- 未命中 -> 返回「未识别」
```

### 3.2 语音识别与自然语言理解

```javascript
function handleVoiceQuery(text) {
  // 支持的自然语言模式
  var patterns = [
    /(.+?)是(?:属于|算)?什么垃圾/,
    /(.+?)属于(?:什么|哪类)垃圾/,
    /(.+?)应该扔(?:到|在)?(?:哪个|什么)垃圾桶/,
    /(.+?)怎么(?:分类|扔|处理)/,
  ];
  // 正则提取物品名 -> 查数据库 -> 显示 + TTS
}
```

### 3.3 数据库设计

内置 150+ 常见生活垃圾，按四分类组织：

- **可回收物** (38项): 废纸、塑料瓶、易拉罐、玻璃瓶、旧衣服等
- **有害垃圾** (27项): 废电池、废灯管、过期药品、废油漆等
- **厨余垃圾** (40项): 剩菜剩饭、果皮、蛋壳、骨头、茶叶渣等
- **其他垃圾** (65项): 卫生纸、烟蒂、陶瓷碎片、贝壳、大棒骨等

### 3.4 规则引擎设计

60+ 条正则规则覆盖常见关键词，按优先级排序。每条规则包含：正则模式、所属类别、推理依据说明。

### 4.2 局限性分析

| 局限性 | 说明 | 改进方向 |
|-------|------|---------|
| 数据库覆盖有限 | 仅 150+ 物品，偏远区域可能缺失 | 社区众包贡献机制 |
| 规则引擎准确率 | 规则匹配可能产生歧义 | 引入 BERT 分类模型 |
| 语音识别依赖浏览器 | 仅 Chrome/Edge 支持 | 备用第三方 ASR API |
| 封闭数据集 | 静态 JSON，无自动更新 | 定期爬取官方数据 |
| 无用户反馈 | 不记录用户纠错 | 增加反馈按钮 |
| 单机知识库 | 无法跨设备共享 | 云端 + 本地双模式 |

## 5. 个人收获与改进方向

### 5.1 个人收获

1. **Web Speech API 实战**: 深入了解了浏览器原生语音识别与合成能力。webkitSpeechRecognition 的接口设计、语言模型选择、错误处理等。
2. **中国垃圾分类标准梳理**: 通过构建数据库，系统梳理了四分类法的具体规则，对「粽叶为什么不是厨余垃圾」「大棒骨与鸡骨头的分类差异」等细节有了清晰认知。
3. **规则引擎设计**: 在精确/模糊匹配失效时，规则引擎作为兜底方案发挥了关键作用。
4. **自然语言理解简化**: 在没有 NLP 库的情况下，通过正则组合覆盖常见的用户问句模式。

### 5.2 改进方向

1. **机器学习分类器**: 使用微调后的 BERT-Chinese 或 TF-IDF + SVM 分类器。
2. **图像识别**: 增加拍照识别功能，使用 MobileNet + TensorFlow.js。
3. **多模态交互**: 结合语音 + 图像 + 文字三种输入方式。
4. **社区共建数据库**: 引入用户纠错和补充机制。
5. **离线模型**: 使用 ONNX Runtime 在浏览器端运行轻量分类模型。
6. **多城市适配**: 支持城市级分类配置差异。
